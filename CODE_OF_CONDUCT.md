# Code of Conduct

## Our Pledge

We are committed to providing a welcoming and inclusive environment for everyone who wants to learn, contribute, and engage with this Node.js Production Playbook.

This repository exists to help engineers grow their skills and understanding. We expect all participants to contribute to a respectful, constructive, and professional community.

## Our Standards

### Positive Behavior

- **Be respectful**: Treat everyone with respect, regardless of their experience level
- **Be constructive**: Provide helpful feedback that improves the content
- **Be professional**: Maintain a professional tone in all interactions
- **Be patient**: Remember that everyone is learning and growing
- **Be collaborative**: Work together to improve the quality of the playbook

### Unacceptable Behavior

- Harassment, discrimination, or personal attacks of any kind
- Trolling, insulting comments, or deliberately derailing discussions
- Publishing others' private information without permission
- Spam, promotional content, or off-topic discussions
- Any conduct that would be inappropriate in a professional setting

## Content Standards

When contributing or discussing content:

- **Focus on technical merit**: Critique ideas and code, not people
- **Provide evidence**: Back up claims with documentation or real-world experience
- **Acknowledge uncertainty**: It's okay to say "I don't know" or "I'm not sure"
- **Respect different perspectives**: There are often multiple valid approaches
- **Stay on topic**: Keep discussions relevant to Node.js and production systems

## Enforcement

### Reporting

If you experience or witness unacceptable behavior:

1. Report it by opening a GitHub issue (if appropriate) or contacting the repository maintainer
2. Provide details about the incident
3. All reports will be reviewed and investigated

### Consequences

Violations may result in:

- Warning and request to modify behavior
- Temporary ban from participating in discussions
- Permanent ban from the repository

The severity of consequences will match the severity of the violation.

## Scope

This Code of Conduct applies to:

- All repository spaces (issues, pull requests, discussions)
- Public spaces when representing the project
- Private communications related to the project

## Attribution

This Code of Conduct is adapted from standard open-source community guidelines, tailored for an educational technical repository.

## Questions

If you have questions about this Code of Conduct, please open a GitHub Discussion.

**Remember**: This playbook exists to help engineers learn and grow. Let's maintain an environment where everyone feels comfortable asking questions, sharing knowledge, and improving their skills.
